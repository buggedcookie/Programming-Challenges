# 📝 Challenges List

### Definition of Fields

| Difficulty | Time Estimate         | Lines of Code | Concepts Focus                                    |
| ---------- | --------------------- | ------------- | ------------------------------------------------- |
| 🟢 Easy    | ~30 min to 2 hours    | ~10 to 50     | Basic I/O, Loops, Conditionals, Simple Structures |
| 🟡 Medium  | ~2 hours to 1 day     | ~50 to 200    | OOP, Simple Algorithms, File I/O, Error Handling  |
| 🔴 Hard    | 1 day to several days | 200+          | Advanced Algorithms, Networking, Concurrency      |

| Field             | Description                                                                                                |
| ----------------- | ---------------------------------------------------------------------------------------------------------- |
| Number            | Incremental based on when the challenge was added (01, 02, 03...). Always 2 digits.                        |
| Difficulty        | Represented visually with an emoji: 🟢 Easy, 🟡 Medium, 🔴 Hard.                                           |
| Name              | CamelCase short name of the challenge. Keep it short, clear, and consistent. No underscores or spaces.     |
| Short Description | A very short (1 sentence) description explaining the purpose of the challenge. Should be clear and direct. |
| Link              | Relative link to the challenge's README.md file.                                                           |

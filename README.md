# Programming Challenges Repository

This repository contains a collection of programming challenges aimed at helping both beginners and experienced developers improve their skills through practice.

## Purpose

This repo is for **learning**. It provides a full list of programming challenges in various languages, categorized by type and difficulty. Anyone can contribute by adding new challenges or improving existing ones. The goal is to keep challenges manageable, understandable, and useful for learning.

---

## Folder Structure

The structure of this repository is as follows:

```
Repos > Challenges > Programming Language > Challenge Type > Challenge
```

- **Repos**: The root folder of the repository.

- **Programming Language**: Folders for different programming languages (e.g., CSharp, CPlusPlus).

- **Challenge Type**: Challenges grouped by their type (e.g., **Apps**, **Games**, **100Lines**).

- **Challenge**: Each individual challenge in its own folder under the appropriate challenge type.

### Example:

```
Challenges > CSharp > Games > 01_SnakeGUI
```

In this example, **SnakeGUI** is a challenge within the **Games** category for C#.

---

## Challenge Naming Convention

Challenges should be named following this format:

```
01_ChallengeName
```

For example:

```
01_SnakeGUI
02_SnakeConsole
```

This naming structure ensures everything stays organized and easily trackable.

---

## How to Contribute

To add a new challenge or contribute to the repository, please refer to the [CONTRIBUTING.md](CONTRIBUTING.md) for detailed instructions on how to submit challenges, add new categories, and follow proper branch protocols.

**Important**: If you're adding a small variation of an existing challenge (e.g., adding a second snake to **SnakeGUI** or **SnakeConsole**), it's better to modify the original challenge rather than creating a new one. Keep the changes simple and make sure the original code still works.

---

### Main Branch

The **Main** branch contains the foundational, clean challenges.

- **Purpose**: Focused on keeping challenges simple and beginner-friendly.

- **Contributions**: Pull requests should only add new challenges. Variations should remain minimal unless they significantly enhance the learning experience.

### Experimental Branch

The **Experimental** branch is for more complex or experimental features like AI, multiplayer, or cheat codes.

- **Purpose**: Allows challenges to evolve while keeping the **Main** branch focused on core challenges.

- **Contributions**: If your modifications are more advanced, like adding AI or multiplayer modes, submit them to the **Experimental** branch.

Feel free to experiment in your own fork. If you’re contributing to the **Experimental** branch, ensure your modifications enhance the challenge but don’t (over) complicate it for beginners.

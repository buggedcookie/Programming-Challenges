# Contributing Guide

Thanks for your interest in improving the Programming Challenges Repository!  
Please follow these simple rules when contributing:

---

## How to Contribute

1. **Fork** the repository.

2. **Create a new folder** inside the correct language and challenge type (Apps, Games, 100Lines).

3. **Name your folder properly**:  
   Format:

```
##_ChallengeName
```

Example:

```
01_SnakeConsole
02_SnakeGUI
03_TicTacToe
04_Chess
```

4. **Inside your folder**, include:
- A `README.md` or `Instructions.md` explaining:
  
  - Rules
  
  - Expected result
  
  - Test cases

- All your source code files.

- No pre-compiled stuff, everything must be open

---

## Rules

- Keep challenges **simple and clean**.

- No bloated files (`bin/`, `obj/`, `.exe`, etc.).

- Everything must be easily editable/readable.

- Anything suspicious/hard-to-read will be **refused**.

- Code must be as **self-contained** as possible (easy to compile/run).

- Use clear folder and file names.

- Stick to the **naming conventions**.

- **One challenge = One folder**.

> ⚠️ **Important:**
> Any attempt to submit malicious code (e.g., malware, viruses, harmful scripts) will result in a **permanent ban** from this repository and may be reported to GitHub for further action.

---

## 📂 About Branches

- If your contribution is a **basic challenge** or a **small project**, submit it to the **Main** branch.

- If it’s a **complex or experimental feature** (such as AI, multiplayer modes, or large upgrades), submit it to the **Experimental** branch.

- If it’s a **simple modification** (e.g., bug fixes, readability improvements, minor performance boosts) to an existing challenge, submit it to the **Main** branch.

> **Note**: 
> The **Main** branch is stricter. It contains the **core, beginner-friendly challenges**. 
> The **Experimental** branch accepts **advanced or enhanced versions** of challenges (e.g., multiplayer Snake).

---

## 📁 Adding a New Category

You can propose a **new category** in two ways:

- A **new programming language folder** (e.g., adding Python, Rust, etc.).

- A **new challenge type folder** inside an existing language (e.g., adding a new challenge type not fitting in Apps, Games, or 100Lines).

### How to propose it:

1. Open a **Pull Request** to the appropriate branch (**Main** or **Experimental**).

2. Clearly **explain why** the new category is needed.

3. (Optional but recommended) Provide an **example challenge** to justify the addition.

## 🔀 Proposing a New Hierarchy

If you believe the current hierarchy needs improvement or think there is a better way to organize the repository:

1. **Propose a new hierarchy structure** for organizing challenges, languages, and categories.

2. Provide **clear reasoning** on why the new structure is better for long-term maintainability, scalability, and clarity.

3. Explain how it **improves organization** and **keeps the repository clean** and easy to navigate, especially as the repository grows.

> **Note**:  
> Any proposed changes to the hierarchy should focus on **enhancing the learning experience** and **ensuring the repository remains easy to manage** in the future.  
> Make sure the new structure is intuitive and easy for new contributors to understand.

---

## Final Note

Keep it clean, organized, and beginner-friendly.
This repo is for **learning**, not showing off.
